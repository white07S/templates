# Configuration Guide

Learn how to configure your development environment for optimal performance and productivity.

## Configuration File

The main configuration file is located at the root of your project: `config.json`

### Basic Configuration

Here's a minimal configuration to get you started:

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "environment": "development",
  "port": 3000,
  "database": {
    "type": "postgresql",
    "host": "localhost",
    "port": 5432,
    "name": "myapp"
  }
}
```

## Environment Variables

You can use environment variables to override configuration values:

```bash
# .env file
NODE_ENV=production
PORT=8080
DATABASE_URL=postgresql://user:pass@localhost:5432/myapp
API_KEY=your-secret-api-key
```

### Loading Environment Variables

```javascript
// Load environment variables
require('dotenv').config();

const config = {
  port: process.env.PORT || 3000,
  database: process.env.DATABASE_URL,
  apiKey: process.env.API_KEY
};
```

## Development Configuration

For development environments, use these recommended settings:

```json
{
  "development": {
    "debug": true,
    "verbose": true,
    "hot-reload": true,
    "source-maps": true,
    "cache": false,
    "minify": false
  }
}
```

## Production Configuration

For production deployments:

```json
{
  "production": {
    "debug": false,
    "verbose": false,
    "hot-reload": false,
    "source-maps": false,
    "cache": true,
    "minify": true,
    "compression": "gzip",
    "clustering": true
  }
}
```

## Database Configuration

### PostgreSQL

```json
{
  "database": {
    "type": "postgresql",
    "host": "localhost",
    "port": 5432,
    "username": "dbuser",
    "password": "dbpass",
    "database": "myapp",
    "ssl": true,
    "pool": {
      "min": 2,
      "max": 10
    }
  }
}
```

### MongoDB

```json
{
  "database": {
    "type": "mongodb",
    "url": "mongodb://localhost:27017",
    "database": "myapp",
    "options": {
      "useUnifiedTopology": true,
      "useNewUrlParser": true
    }
  }
}
```

## API Configuration

Configure external APIs and services:

```json
{
  "apis": {
    "stripe": {
      "publishableKey": "pk_test_...",
      "secretKey": "sk_test_..."
    },
    "sendgrid": {
      "apiKey": "SG...."
    },
    "aws": {
      "region": "us-east-1",
      "accessKeyId": "AKIA...",
      "secretAccessKey": "..."
    }
  }
}
```

## Logging Configuration

Set up logging levels and outputs:

```json
{
  "logging": {
    "level": "info",
    "format": "json",
    "outputs": [
      {
        "type": "console",
        "level": "debug"
      },
      {
        "type": "file",
        "path": "./logs/app.log",
        "level": "error",
        "rotate": {
          "maxSize": "10m",
          "maxFiles": 5
        }
      }
    ]
  }
}
```

## Security Configuration

Important security settings:

```json
{
  "security": {
    "cors": {
      "origin": ["https://example.com"],
      "credentials": true
    },
    "helmet": {
      "contentSecurityPolicy": true,
      "hsts": {
        "maxAge": 31536000,
        "includeSubDomains": true
      }
    },
    "rateLimit": {
      "windowMs": 900000,
      "max": 100
    }
  }
}
```

## Performance Optimization

### Caching

```json
{
  "cache": {
    "enabled": true,
    "type": "redis",
    "redis": {
      "host": "localhost",
      "port": 6379,
      "ttl": 3600
    }
  }
}
```

### CDN Configuration

```json
{
  "cdn": {
    "enabled": true,
    "provider": "cloudflare",
    "domain": "cdn.example.com",
    "assets": [
      "/images",
      "/styles",
      "/scripts"
    ]
  }
}
```

## Validation

Use our CLI to validate your configuration:

```bash
# Validate configuration
our-product config validate

# Test database connection
our-product config test-db

# Show current configuration
our-product config show
```
